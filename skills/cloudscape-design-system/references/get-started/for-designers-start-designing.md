# Start designing

Welcome to Cloudscape. Let's start building. Here, you'll find links to our design library, visual foundation, and information about our components, patterns, and demos.

## Explore the system

## Explore our components

Each of our [component articles](/components/index.html.md) offers usage guidelines, a comprehensive list of the component's features, design do's and dont's, writing guidelines and accessibility guidelines. Teams can use the playground to test out how components look and behave, even if you have no coding experience.

## Check out our patterns

Our [patterns](/patterns/index.html.md) provide design guidance on how to assemble common flows and solutions out of our components. Our pattern articles clarify key UX concepts, and provide writing and accessibility guidelines for creating solutions.

Use our design patterns as a base for common design solutions, to easily maintain consistency.

## See our demos

Our [demos](/demos/index.html.md) show how our components and patterns can be arranged into ready-made flows and pages.

You can use them as reference for how to build consistent pages with Cloudscape.

Our visual [foundation](/foundation/index.html.md) consists of the core elements of the system, such as [typography](/foundation/visual-foundation/typography/index.html.md) , [iconography](/foundation/visual-foundation/iconography/index.html.md) , and [colors.](/foundation/visual-foundation/colors/index.html.md)

## Explore design resources

The [Figma component library](/get-started/for-designers/design-resources/index.html.md) contains all components and styles within one single file. The component library file can be used both as a library (which you can [publish](https://help.figma.com/hc/en-us/articles/360025508373-Publish-styles-and-components) to access all components from any design file), and a sticker sheet (to copy and paste elements to your design files). For more details about the Cloudscape Figma library, check out the getting started guide inside the library file.
