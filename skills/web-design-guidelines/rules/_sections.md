# Sections

This file defines all sections, their ordering, impact levels, and descriptions.
The section ID (in parentheses) is the filename prefix used to group rules.

---

## 1. Accessibility (accessibility)

**Impact:** CRITICAL  
**Description:** Accessibility is fundamental to inclusive design. These rules ensure web content is perceivable, operable, understandable, and robust for all users, including those using assistive technologies.

## 2. Layout (layout)

**Impact:** HIGH  
**Description:** Proper layout creates visual hierarchy and guides users through content. These rules cover spacing, alignment, and structural organization.

## 3. Typography (typography)

**Impact:** HIGH  
**Description:** Typography affects readability and user experience. These rules ensure text is legible, properly sized, and appropriately styled.

## 4. Color (color)

**Impact:** MEDIUM-HIGH  
**Description:** Color conveys meaning and creates visual appeal. These rules ensure proper contrast, consistent usage, and accessibility compliance.

## 5. Interaction (interaction)

**Impact:** MEDIUM-HIGH  
**Description:** Interaction patterns affect usability and user satisfaction. These rules cover buttons, forms, navigation, and feedback mechanisms.

## 6. Responsive Design (responsive)

**Impact:** MEDIUM  
**Description:** Responsive design ensures interfaces work across all device sizes. These rules cover breakpoints, fluid layouts, and adaptive patterns.

## 7. Performance (performance)

**Impact:** MEDIUM  
**Description:** Visual performance affects perceived speed and user experience. These rules cover image optimization, animations, and rendering efficiency.

## 8. Animation & Motion (animation)

**Impact:** LOW-MEDIUM  
**Description:** Animation enhances user experience when used appropriately. These rules cover timing, easing, and motion preferences.
