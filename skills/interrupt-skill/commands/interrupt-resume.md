# /interrupt-resume — 恢复中断任务

用户参数: $ARGUMENTS

## 前置检查

检查 `~/w/interrupts/` 是否存在且是 git 仓库。如果不是，提示用户先运行 `/interrupt-init`，然后停止。

## Step 1: 选择记录

先列出 `~/w/interrupts/active/` 下所有 `.md` 文件（排除 .gitkeep）。

- **active/ 为空** → 告诉用户"没有待恢复的中断记录"，停止。
- **有 $ARGUMENTS** → 在文件名和文件内容中 grep 匹配关键词，列出匹配结果。
  - 有唯一匹配 → 直接使用
  - 多个匹配 → 列出让用户选
  - 无匹配 → 列出全部让用户选
- **无 $ARGUMENTS** → 列出全部记录，用 AskUserQuestion 让用户选一条。

## Step 2: 读取记录

用 Read 工具读取选中的中断记录文件，理解其内容。

## Step 3: 苏格拉底探测（1-3 轮）

根据记录内容，用 AskUserQuestion 逐步探测用户当前的理解状态。目的是区分"还记得"和"已经忘了"。

探测策略：

**第一轮**（必问）：围绕"停在哪"：
- "你上次停在 {停在}，你还记得当时卡在哪吗？还是需要我帮你回顾？"

**第二轮**（按需）：围绕"下一步"或"未确认"：
- 如有下一步："你说下一步是 {next_step}，你现在知道怎么做了吗？"
- 如有未确认："上次未确认的问题：{unresolved}，你有新的想法吗？"

**第三轮**（按需）：围绕"已确认"中可能遗忘的点：
- "你之前确认了 {confirmed}，这些结论你还认同吗？"

**提前结束条件**：
- 用户回答表明记忆清晰 → 跳过后续探测
- 用户说"我全忘了"或"直接开始" → 跳过探测，AI 从记录中总结要点后继续

## Step 4: 恢复摘要（对话内）

在对话中输出简要的恢复摘要，帮用户和自己对齐上下文：

```
**恢复摘要**：
- 任务：{任务名}
- 上次停在：{停在}
- 当前状态：{基于探测的理解}
- 接下来：{下一步行动}
```

## Step 5: 归档 + Git 同步

```bash
cd ~/w/interrupts && git pull --rebase 2>/dev/null; mv "active/{文件名}" "archived/" && git add active/ archived/ && git commit -m "resume: {任务名简述}" && git push 2>/dev/null || true
```

## Step 6: 就地继续

告诉用户"已恢复，我们继续。"然后直接进入工作状态——根据"下一步"和探测结果，开始帮用户执行任务。
